# نشر موقع MistVil على استضافة Hostinger

موقعك يحتاج شيئين ليعمل ويرى **جميع الزوار** الروايات المنشورة:

1. **الملفات المبنية** (`dist/`) — وليس ملفات المشروع الخام. ملف `index.html` في جذر
   المشروع مخصص للتطوير فقط ولا يعمل مباشرة على الاستضافة.
2. **قاعدة بيانات مشتركة** — تعمل الآن عبر PHP (`api/db.php`) بدون الحاجة إلى تشغيل
   Node.js، وهذا مدعوم تلقائياً على استضافة Hostinger المشتركة.

عند بناء المشروع، ينتج مجلد `dist/` يحتوي على: الموقع المبني + `api/db.php` + ملف
`.htaccess`. رفع محتويات هذا المجلد إلى `public_html` هو كل المطلوب.

---

## الطريقة الموصى بها: نشر تلقائي عبر GitHub Actions (FTP)

أضفتُ ملف `.github/workflows/deploy-hostinger.yml` الذي يبني الموقع ويرفعه تلقائياً
إلى Hostinger عند كل تحديث على فرع `main`. لتفعيله مرة واحدة:

### 1) احصل على بيانات FTP من Hostinger
- ادخل hPanel → **Files → FTP Accounts**.
- ستجد: **FTP hostname/server** (مثل `ftp.yourdomain.com` أو رقم IP)، و**username**،
  و**password** (يمكنك إنشاء/تغيير كلمة المرور من هناك).

### 2) أضف الأسرار في GitHub
- افتح مستودعك على GitHub → **Settings → Secrets and variables → Actions →
  New repository secret**، وأضف ثلاثة أسرار:
  - `FTP_SERVER` = عنوان خادم FTP
  - `FTP_USERNAME` = اسم مستخدم FTP
  - `FTP_PASSWORD` = كلمة مرور FTP

### 3) تحقق من مجلد الوجهة
- في ملف `deploy-hostinger.yml`، القيمة الافتراضية `server-dir: /public_html/` تناسب
  النطاق الرئيسي. إن كنت تستخدم نطاقاً فرعياً أو نطاقاً مضافاً، غيّرها إلى مجلد ذلك
  النطاق (مثل `/domains/yourdomain.com/public_html/`).

### 4) شغّل النشر
- ادفع أي تحديث إلى `main` (أو من تبويب **Actions** اختر الـ workflow واضغط
  **Run workflow**). سيبني ويرفع الموقع خلال دقيقتين.

بعدها: أي رواية ينشرها المالك تظهر فوراً لكل الزوار، ولا تُمسح عند إعادة النشر
(ملف قاعدة البيانات `mistvil_db.json` مستثنى من الحذف).

## طريقة بديلة: رفع يدوي مرة واحدة

### الخيار الأسهل: تنزيل النسخة الجاهزة من GitHub (بدون تثبيت أي شيء)
عند كل تحديث على `main`، يبني GitHub الموقع تلقائياً ويرفق النتيجة كملف قابل
للتنزيل حتى لو لم تُضبط أسرار FTP:

1. افتح المستودع على GitHub → تبويب **Actions** → آخر تشغيل لـ
   **Deploy to Hostinger**.
2. في أسفل صفحة التشغيل، قسم **Artifacts** → نزّل **mistvil-site**.
3. فك الضغط وارفع **محتويات** المجلد إلى `public_html` عبر
   **hPanel → File Manager**، مع تفعيل إظهار الملفات المخفية لرفع
   `.htaccess` و`api/.user.ini` أيضاً.

### أو البناء على جهازك:

1. على جهازك: `npm install` ثم `npm run build`.
2. افتح مجلد `dist/` الناتج.
3. ارفع **محتويات** `dist/` (وليس المجلد نفسه) إلى `public_html` عبر
   **hPanel → File Manager** أو أي برنامج FTP. تأكد من رفع الملفات المخفية أيضاً
   (`.htaccess`).
4. تأكد أن مجلد `api/` قابل للكتابة (الصلاحيات 755 عادةً كافية) ليتمكن `db.php` من
   إنشاء `mistvil_db.json`.

## بعد النشر

- **أعد نشر رواياتك مرة واحدة** بعد وصول التحديث، لأن قاعدة البيانات القديمة ربما
  فُقدت سابقاً بسبب الخلل الذي أصلحناه. بعد ذلك تبقى الروايات ظاهرة للجميع بشكل دائم.
- للتحقق السريع أن قاعدة البيانات تعمل: افتح `https://yourdomain.com/api/db` في
  المتصفح — يجب أن ترى بيانات JSON (مثل `{}` أو قائمة رواياتك)، وليس صفحة خطأ.

## ملاحظة أمنية
هذا النموذج يخزّن المحتوى في ملف مشترك على الخادم عبر PHP، والحسابات وكلمات المرور
تبقى محلية في متصفح كل مستخدم ولا تُرفع للخادم. الحماية الكاملة (تسجيل دخول حقيقي
بجلسات على الخادم) تتطلب تطويراً إضافياً يمكن تنفيذه لاحقاً عند الحاجة.
